Getting Started with Bricks v2.0.0 BETA

Disclaimer: While you see the BETA in the Bricks version name above, just be warned that this version is constantly being updated without having the version and URL to the CSS file changed. For this reason it is highly advised to either use one of the earlier versions of Bricks or PennyPixels, or copy the current CSS to your project instead of linking to it directly. Another really good reason to do it this way is that the hosting currently is not a CDN and will likely be changing locations soon.

Add CSS

Add the following Bricks CSS to your head tag. This already incorporates Normalize.css and Bootstrap v4.0.0-alpha.6 so you should not add them separately. Note: It is a good idea to download this file and link to it within your own app instead of linking directly for better performance since we are not hosting on a CDN. If you'd prefer you can download our SASS files to use instead (you'll want the index file in addition to the contents of the bricks directory, though you may have to customize the paths depending on your file structure).

Add Javascript

Add Bootstrap's JavaScript plugins, jQuery, and Tether near the end of your pages, right before the closing body tag. Be sure to place jQuery and Tether first, as Bootstrap code depends on them. While we've suggested jQuery's slim build here, the full version and versions later than 3.1.1 are also supported.

Copy

Add Web Fonts

For the best page loading performance, load Open Sans from Google Fonts asynchronously by adding the following snippet immediately after your opening head tag. Also, add class="wf-loading" to your opening html tag to avoid a FOUT. Also be sure to add FontAwesome since some of our components rely on them. Note, because Material Icons do not have a fall-back font and we want to avoid a flash of the icon name while they load, we will load those synchronously.
